# Output Patterns

## 事实不清时必须先问

```markdown
任务类型：...

Step 1: 【案情研判与多模态解析】
当前缺少以下关键事实，暂不能直接出正式方案：
1. ...
2. ...
3. ...
4. ...
5. ...
```

问完即停，不进入 Step 2 到 Step 4。

## 合同类四段式

```markdown
任务类型：合同起草 / 合同审核 / 合同修改

Step 1: 【案情研判与多模态解析】
- 关键事实：
- 法律适用：
- 调用模块：
- 采用策略：
- 若为合同修改，先写明：`本次采用【Level 1 / Level 2 / Level 3】策略，理由是……`

Step 2: 【法律成果交付】
1. 红黑修订版 / 关键新增条款清单
2. 最终洁净版
3. 合同修改对比说明表
4. 合同审核时固定栏目：
   - 【全局风险定级】
   - 【权利不对等分析】
   - 【必备条款缺失/瑕疵】
   - 【违约与救济陷阱】
   - 【程序与合规隐患】
   - 【谈判筹码与反制策略】

Step 3: 【风险阻断与行动指南】
1. ...
2. ...

Step 4: 【权威法源与实证清单】
1. 现行有效法条
2. 最高院裁判规则与司法解释
3. 类案同判指引
4. 必要时补充：实证来源台账
```

## 诉讼与仲裁类四段式

```markdown
任务类型：诉讼/仲裁文书 / 代理方案 / 证据与程序

Step 1: 【案情研判与多模态解析】
- 关键事实：
- 程序阶段：
- 我方目标：
- 核心争点：
- 调用模块：

Step 2: 【法律成果交付】
1. 起诉状 / 答辩状 / 代理词 / 代理方案正文
2. 证据目录或举证矩阵

Step 3: 【风险阻断与行动指南】
1. 证据补强
2. 程序动作
3. 对方抗辩预判

Step 4: 【权威法源与实证清单】
1. 现行有效法条
2. 最高院裁判规则与司法解释
3. 类案同判指引
4. 必要时补充：实证来源台账
```

## OCR 证据与图片识别四段式

```markdown
任务类型：图像与电子证据识别

Step 1: 【案情研判与多模态解析】
- 图片来源：
- 识别目标：
- 可读性与疑点：
- 调用模块：

Step 2: 【法律成果交付】
1. 关键内容提取
2. 证据时间轴
3. 证据目录
4. 质证意见 / 伪造疑点清单

Step 3: 【风险阻断与行动指南】
1. 原图补取
2. 取证留痕
3. 公证 / 保全 / 设备核验建议

Step 4: 【权威法源与实证清单】
1. 证据规则
2. 最高院司法解释与裁判规则
3. 类案同判指引
4. 图片来源与保全台账
```

```markdown

Step 1: 【案情研判与多模态解析】
- 目标：
- 数据或材料范围：

1. 测算结果 / 风险地图 / 检索报告
2. 计算公式 / 命中条款 / 公开来源台账

Step 3: 【风险阻断与行动指南】
1. ...
2. ...

Step 4: 【权威法源与实证清单】
1. 现行有效法条
2. 最高院裁判规则与司法解释
3. 类案同判指引

```
