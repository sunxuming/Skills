# Data Asset Playbook

## 核心任务

- 数据来源合法性审查
- 授权链和权属控制审查
- 数据资源入表前置条件审查
- 数据交易、质押、ABS 底层资产穿透审查
- 个人信息、重要数据、匿名化、脱敏方案合规核查

## 默认审查顺序

1. 识别数据类型
- 业务数据
- 个人信息
- 敏感个人信息
- 重要数据

2. 穿透来源
- 自有采集
- 第三方购买
- API 调用
- 抓取或爬虫
- 合作方共享

3. 穿透授权链
- 用户授权
- 合作协议
- 平台协议
- 数据处理规则
- 隐私政策

4. 检查可控制性
- 是否具备占有、使用、收益、管理上的可控制性
- 是否存在被第三方随时撤销、终止、封禁的风险

5. 检查入表条件
- 是否满足确认条件
- 是否可以可靠计量
- 是否存在重大合规瑕疵
- 是否存在无法持续控制的问题

## 必查风险

- 用爬虫抓取但无充分授权
- API 使用超出协议边界
- 数据集中含有未脱敏个人信息
- 脱敏无法逆推验证
- 数据处理目的与原始授权不一致
- 交易结构中权利边界不清
- 资产看似可入表，实则无法稳定控制或收益

## 输出建议

- 合规结论
- 主要风险点
- 入表可行性判断
- 必要整改清单
- 交易结构优化建议
- Step 4 法源与类案支撑清单
